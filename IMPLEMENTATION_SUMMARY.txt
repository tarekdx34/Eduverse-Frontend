API ENDPOINTS USED
==================

Endpoint 1: GET /api/assignments
───────────────────────────────
Returns flat array of assignments (NOT paginated)

Response Format:
{
  "id": "38",
  "courseId": "14",
  "title": "Lab 1 : Test",
  "description": "Testing tabs",
  "maxScore": "100.00",
  "dueDate": "2026-03-28T00:00:00.000Z",
  "status": "published",
  "submissionType": "file",
  "course": {
    "id": "14",
    "name": "Arabic Language Skills",
    "code": "AH102"
  }
}

Endpoint 2: GET /api/assignments/:id/submissions/my
──────────────────────────────────────────────────
Returns student's submission for an assignment

Response Format:
{
  "id": "123",
  "submissionStatus": "submitted|graded",
  "submittedAt": "2026-03-27T10:30:00.000Z",
  "score": "85.50",
  "feedback": "Great work!"
}

Returns: 404 Not Found (handled gracefully as "not-submitted")

PERFORMANCE
===========

API Calls per Page Load:
  • 1x GET /api/assignments (fetch list)
  • N x GET /api/.../submissions/my (fetch statuses)
  • All status requests run in PARALLEL (not sequential)

Optimization:
  • Promise.all() for concurrent requests
  • useApi hook may cache results
  • No blocking operations
  • Graceful fallback if API slow

Expected Performance:
  • Initial page load: 1-2 seconds (includes API calls)
  • Submission status update: Real-time via form submission
  • No additional requests after initial load
